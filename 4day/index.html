Session 4.2

성능 테스트 및
최적화 방안

VPC 환경에서 최고의 성능 끌어내기

성능 벤치마킹 도구

nGrinder

네이버에서 개발한 부하 테스트 플랫폼. 스크립트 기반의 시나리오 작성 및 분산 테스트 환경을 지원하여 대규모 테스트에 용이합니다.

JMeter

Apache 재단의 오픈소스 도구. GUI 기반으로 테스트 계획을 수립할 수 있어 직관적이며, 다양한 프로토콜을 지원합니다.

최적화는 순환 과정입니다

1. 성능 측정

부하 테스트 수행

2. 병목 구간 식별

결과 데이터 분석

3. 최적화 적용

서버 스펙, 아키텍처 변경

4. 효과 검증

다시 1번부터 반복

Session 4.3

마이그레이션 이슈 & 트러블슈팅

사례를 통해 배우는 문제 해결

주요 마이그레이션 문제 유형 (사례 연구)

네트워크

잘못된 ACG/NACL 설정, 라우팅 테이블 오류로 인한 통신 불가

호환성

OS 커널 버전, 라이브러리 의존성 차이로 인한 애플리케이션 실행 실패

데이터 정합성

데이터 이전 중 누락 발생, CDC 적용 실패로 인한 데이터 불일치

로그 분석 기반 트러블슈팅

# tail -f /var/log/httpd/error_log

[timestamp] [mpm_prefork:notice] [pid 1234] AH00163: Apache/2.4.6 (CentOS) configured -- resuming normal operations

[timestamp] [core:error] [pid 5678] [client 10.0.1.10:12345] Connection refused: AH00957: HTTP: attempt to connect to 10.0.2.20:8080 (was-server) failed

# 분석: Web 서버(10.0.1.10)가 WAS 서버(10.0.2.20)의 8080 포트로 접속을 거부당함.

> 의심 원인: WAS 서버의 ACG 또는 NACL 정책 확인 필요

Session 4.4

PaaS를 활용한 배포 및 확장

Cloud Native 방식으로 서비스 운영하기

네이버클라우드 주요 PaaS 상품

Cloud Functions

서버리스 컴퓨팅 서비스. 코드만 업로드하면 인프라 관리 없이 실행 가능. 이벤트 기반의 간단한 작업 처리에 적합합니다.

Kubernetes Service

컨테이너화된 애플리케이션을 자동으로 배포, 확장 및 관리하는 서비스. MSA와 같이 복잡하고 확장성 높은 애플리케이션에 적합합니다.
